🎯 Cele Projektu

  • Techniczny: Budowa responsywnego interfejsu typu Dashboard z bocznym menu.
  • Funkcjonalny: Stworzenie systemu zapisu danych użytkownika w pamięci lokalnej (Local Storage).
  • Rozrywkowy: Integracja interaktywnych map gier oraz implementacja gry z AI.
  • Estetyczny: Wdrożenie płynnego systemu zmiany motywów (Dark/Light Mode).

    Prezentacja Projektu

    1. Temat i Planowanie

    • Temat: Multi-Tool Dashboard – zintegrowane centrum narzędzi dla gracza.
    • Plan: Projekt został podzielony na fazy (Analiza -> Szkielet -> Logika -> Testy), co pozwoliło na płynne dodawanie nowych funkcji, jak dynamiczne tła.

    2. Struktura i Stylizacja

    • HTML5: Użycie semantycznych tagów (nav, main, section) dla lepszej struktury.
    • CSS3: Wykorzystanie zmiennych (:root) do łatwego zarządzania kolorami oraz systemu Dark Mode.

    3. Responsywność (RWD)

    • Zastosowanie Media Queries oraz Flexboxa sprawia, że dashboard jest czytelny zarówno na komputerze, jak i na smartfonie (np. zmniejszająca się mapa Elden Ring).

    4. Interakcja z JS

    • DOM: Dynamiczne przełączanie sekcji bez odświeżania strony.
    • LocalStorage: Trwałe zapisywanie listy zakupów oraz wybranego motywu kolorystycznego.
    • Logika: Algorytm losowania cytatów oraz gra "Kółko i Krzyżyk" z prostym AI.

    5. Wyzwania i Doświadczenia

    "Największym wyzwaniem była synchronizacja mapy interaktywnej Elden Ring z nawigacją oraz zapewnienie, że LocalStorage poprawnie wczytuje dane przy starcie strony. Nauczyłem się, jak ważne jest czyste zamykanie tagów HTML przy tak dużej strukturze."

    6. Kontynuacja Rozwoju

    W przyszłości planuję dodać:

    • Zewnętrzne API pogodowe.
    • Większą bibliotekę map z innych gier RPG np. do Wiedźmina 3 lub do Hollow Knight
